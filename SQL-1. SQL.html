Database

데이터베이스는 나중에 검색하고 되찾게 할 수 있도록 데이터를 체계적으로 저장합니다.

하나 이상의 테이블을 포함하고 있으며, 테이블은 행과 열로 구성된다는 점에서 스프레드 시트와 매우 유사합니다.

  • Database / Schema
  • Table
  • 행(Row) / Record
  • 열(Column) / Field


Database Management Systems (DBMS)

데이터베이스 관리 시스템 (DBMS)은 데이터베이스의 생성 및 관리를 가능하게하는 소프트웨어 프로그램입니다.

가장 많이 사용되는 관계형 데이터베이스 관리 시스템은 다음과 같습니다.:

  • Microsoft Access
  • Microsoft SQL Server
  • MySQL (~~> MariaDB)
  • Oracle
  • MongoDB


Structured Query Language (SQL)

SQL은 모든 데이터베이스를 쿼리하는 데 사용되는 언어입니다. 배우기 쉽고, 하는 일이 적은 것처럼 보이지만 성공적인 데이터베이스 애플리케이션의 핵심입니다. SQL을 이해하고 효율적으로 사용하는 것은 효율적인 데이터베이스 응용 프로그램을 설계하는 데 매우 중요합니다.

Data Types


  • Strings - CHAR, VARCHAR, TEXT
  • Date/time - DATE, TIME, YEAR, TIMESTAMP, DATETIME
  • Numbers - DECIMAL (e.g. DECIMAL(5,2): -999.99 ~ 999.99 ), INT 
  • Lists - ENUM, SET
  • https://mariadb.com/kb/en/data-types-numeric-data-types/

NULL 은 데이터베이스에 데이터 값이 없음을 나타 내기 위해 SQL에서 사용되는 특수 마커입니다.

  • NULL 값은 필수(required) 필드에는 사용하지 않습니다.
  • NULL 값은 선택(optional) 필드에 사용합니다.

Basic SQL

  • SQL은 순차적(sequential)이지 않습니다.
  • SQL은 선언적(declarative) 언어 입니다. 방법이 아닌 우리가 원하는 것을 표시합니다.
  • SELECT ... FROM ... WHERE
    • SELECT table_name
    • FROM information_schema.tables
    • WHERE table_schema='mysql'

===================================== (OPTION) 일반 계정 생성 =====

MariaDB [accounts]>CREATE database myuserDB;
MariaDB [accounts]>USE mysql;
MariaDB [accounts]>SELECT host, user, authentication_string FROM user;

MariaDB [accounts]>INSERT user(host, user, password) VALUES('localhost','myuser',password('123456'));
[Note that from MySQL 5.7.6, the user table uses the authentication_string column only to store the password. In addition, it removed the password column.]
MariaDB [accounts]>CREATE USER 'myuser'@'localhost' IDENFIED BY '123456';
MariaDB [accounts]>FLUSH privileges;
MariaDB [accounts]>GRANT all on myuserDB.* to myuser@localhost identified by '123456';
MariaDB [accounts]>SHOW GRANTS FOR myuser@localhost;
MariaDB [accounts]>exit
